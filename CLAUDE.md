## Project Overview

This is the GoodPixel.AI website - a modern static website built with **SvelteKit**, **Tailwind CSS**, and a **neobrutalism design system**. The site is deployed to Firebase Hosting.

## Tech Stack

- **Framework**: SvelteKit (static site generation)
- **Styling**: Tailwind CSS with custom neobrutalism components
- **Language**: TypeScript
- **Deployment**: Firebase Hosting
- **Runtime**: Node.js 22+ (use nvm)

## Development Commands

### Initial Setup

```bash
nvm use 22              # Switch to Node 22
npm install             # Install dependencies
```

### Development

```bash
npm run dev             # Start dev server at http://localhost:5173
npm run dev -- --open   # Start dev server and open browser
```

### Build & Deploy

```bash
npm run build           # Build for production (outputs to build/)
npm run preview         # Preview production build locally
firebase deploy --only hosting  # Deploy to Firebase (login: info@goodpixel.ai)
```

### Other Commands

```bash
npm run check           # Run type checking
npm run check:watch     # Run type checking in watch mode
```

## Architecture

### Directory Structure

- **`src/`** - Source code
  - **`routes/`** - SvelteKit routes (file-based routing)
    - `+page.svelte` - Homepage
    - `+layout.svelte` - Root layout (includes CSS imports and meta tags)
    - `+layout.ts` - Layout configuration (prerender: true)
  - **`lib/`** - Reusable code
    - **`components/`** - Svelte components
      - `BrutalCard.svelte` - Neobrutalism card component
      - `BrutalButton.svelte` - Neobrutalism button component
      - `ServiceCard.svelte` - Service display card
  - `app.css` - Global Tailwind CSS and custom neobrutalism styles

- **`static/`** - Static assets (copied to build/ as-is)
  - `assets/img/` - Images
  - `favicons/` - Favicon files

- **`build/`** - Production build output (generated, not in git)

### Configuration Files

- **`svelte.config.js`** - SvelteKit configuration (uses @sveltejs/adapter-static)
- **`tailwind.config.js`** - Tailwind configuration with neobrutalism colors and shadows
- **`postcss.config.js`** - PostCSS configuration for Tailwind
- **`tsconfig.json`** - TypeScript configuration
- **`vite.config.ts`** - Vite bundler configuration
- **`firebase.json`** - Firebase hosting configuration (public: "build")
- **`.firebaserc`** - Firebase project reference (project: `goodpixel-fbfbe`)

## Neobrutalism Design System

The project uses a custom neobrutalism design system built with Tailwind CSS.

### Color Palette

Defined in `tailwind.config.js`:
- `neo-black`: #000000
- `neo-white`: #FFFFFF
- `neo-yellow`: #FFD700
- `neo-pink`: #FF10F0
- `neo-blue`: #00E0FF
- `neo-green`: #00FF94
- `neo-red`: #FF0051

### Custom Shadows

- `shadow-brutal`: 4px offset shadow
- `shadow-brutal-lg`: 8px offset shadow
- `shadow-brutal-xl`: 12px offset shadow

### Component Classes (in `app.css`)

- `.btn-brutal` - Base brutal button style (yellow)
- `.btn-brutal-pink` - Pink variant
- `.btn-brutal-blue` - Blue variant
- `.card-brutal` - Card with border and shadow
- `.input-brutal` - Form input with brutal styling

### Reusable Components

- **BrutalButton** - Accepts `variant` prop (yellow|pink|blue|green|red), supports both links and buttons
- **BrutalCard** - Card container with optional hover effects
- **ServiceCard** - Specialized card for displaying services with icon, title, and description

## Styling Approach

- Use Tailwind utility classes wherever possible
- Custom neobrutalism utilities are defined in `app.css` using `@layer components`
- All components use bold borders (border-4) and offset shadows for the brutal aesthetic
- Hover effects translate elements and remove shadows for interactive feedback
- Never edit generated CSS files in `build/`

## SvelteKit-Specific Patterns

- **Static Generation**: All pages are prerendered (set in `+layout.ts`)
- **File-based Routing**: Pages in `src/routes/` automatically become routes
- **Component Props**: Use Svelte 5 runes syntax (`$props()`, `{#if}`, `{@render}`)
- **Assets**: Reference static files with absolute paths (e.g., `/assets/img/logo.png`)
